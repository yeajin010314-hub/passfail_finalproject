본 프로젝트는 Spring Boot 기반 계층형 아키텍처로 설계하였으며 Controller, Service, Repository 계층을 분리하여 유지보수성과 확장성을 고려하였습니다.

### Controller

* 요청 데이터 검증
* 응답 반환

### Service

* 비즈니스 로직 처리
* AI 질의응답 처리
* 문제 등록 및 수정 처리
* 문제 검색 및 조회 기능 제공

### Repository (JPA)

* DB 접근 로직 담당
* Entity 기반 데이터 처리
* 문제, 테스트케이스, 태그, AI 채팅 데이터 관리

### Database

# 트러블슈팅 (Troubleshooting)

### AI 상담원 연결 시 모든 관리자에게 동일한 상담이 노출되는 문제

#### 문제

상담 요청이 생성된 이후 특정 관리자가 상담을 수락하더라도 모든 관리자 화면에 동일한 상담이 계속 노출되는 문제가 발생했습니다.

#### 원인

상담 요청 상태만 관리하고 담당 관리자 정보가 저장되지 않아 어떤 관리자가 상담을 처리 중인지 구분할 수 없었습니다.

#### 해결

* AI 채팅 세션에 담당 관리자 ID(assignedAdminId) 추가
* 상담 상태(Enum) 관리 구조 적용
* 상담 수락 시 담당 관리자 정보 저장
* 수락된 상담은 담당 관리자에게만 조회되도록 로직 수정

#### 결과

* 상담 중복 처리 방지
* 관리자 업무 충돌 제거
* 실제 고객센터 형태의 상담 구조 구현
